SECTION 3.1

ربط مفاهيم الذاكرة إلى وحدة التخزين

قبل أن نتعلم الأوامر، يجب أن نفهم أين تعيش التطبيقات.

أين توجد User Space فعلياً؟

في الرسم الهندسي Linux Architecture الذي رسمناه سابقاً، نرى الـ User Space وكأنها تعمل حالياً. ولكن في الواقع (In reality) كافة التطبيقات (مثل الـ Shell) والمكتبات (مثل Glibc) هي عبارة عن ملفات Files مخزنة داخل وحدة التخزين Storage Device مثل الـ Hard Disk أو SD بطاقة.

عندما يقرر الكيرنل تشغيل تطبيق، يستخدم نظام إدارة الملفات File System Management وإدارة الذاكرة Memory Management لنقل هذا الملف الثنائي Binary File من وحدة التخزين إلى الذاكرة العشوائية RAM ثم يوجه المعالج عبر الـ Program Counter لتنفيذ أوامره.

كيف ينتقل التطبيق من التخزين إلى التنفيذ
Storage Device (SD Card / Hard Disk)
وحدة التخزين — التطبيقات مخزنة كملفات ثنائية هنا
↓
File System Management + Memory Management
الكيرنل يدير عملية النقل من التخزين إلى الذاكرة
↓
RAM (Random Access Memory)
الملف الثنائي Binary File يُنسخ إلى الذاكرة
↓
CPU — Program Counter → Execute
المعالج ينفذ الأوامر واحداً تلو الآخر
📘 مفهوم أساسي

التطبيقات في لينكس لا "تعيش" في الذاكرة بشكل دائم. إنها ملفات مخزنة على وحدة التخزين، وعند الحاجة لتشغيلها يقوم الكيرنل بنسخها إلى الـ RAM ثم توجيه المعالج لتنفيذها عبر الـ Program Counter. هذه العملية تتم في كل مرة تشغل فيها أي أمر أو برنامج.

SECTION 3.2

نظام الملفات في لينكس Root File System

كيف يتم تنظيم الملفات في وحدة التخزين؟

الـ File System هو النظام والتصميم الذي يحدد كيف وأين يتم تخزين الملفات والمجلدات.

النظام هيكل الجذور الرمز الفلسفة
Windows توجد عدة جذور أو أقراص مثل C: و D: C:\ , D:\ كل قرص مستقل بجذره الخاص
Linux يوجد فقط واحد جذر تماماً، وتتفرع منه كل الملفات والمجلدات / شجرة واحدة تبدأ من جذر واحد (Slash)
هيكل الـ Root File System في لينكس
/   Root
┌──────────┬──────────┬──────────┬──────────┬──────────┬──────────┐
│   bin   │   sbin  │   lib   │  boot  │  home  │   ...   │
└──────────┴──────────┴──────────┴──────────┴────┬─────┴──────────┘
                                             │
                                           ┌───┴────┐
                                           │techleef│
                                           └────────┘

أهم المجلدات القياسية تحت الجذر /

المسار الاختصار الوظيفة
/bin & /sbin Binaries تحتوي على الملفات التنفيذية (التطبيقات وأوامر النظام الأساسية).
/lib Libraries تحتوي على مكتبات النظام (مثل الـ DLL في ویندوز).
/boot Boot يحتوي على ملفات الإقلاع الضرورية لبدء تشغيل النظام مثل Linux Kernel وBootloader.
/home Home المجلد المخصص لملفات المستخدمين (يعادل Users في ویندوز)، وكل مستخدم له مجلد فرعي بداخله (مثال: /home/techleef).
✅ ماذا يجب أن تتذكر؟

لينكس لا يستخدم الحروف للأقراص (C, D)، بل كل شيء يبدأ من الجذر الأكبر Root Directory وهو / ويستخدم الشرطة المائلة العادية / وليس العكسية \.

SECTION 3.3

بنية الأوامر في لينكس Linux Command Structure

الـ Shell هو مترجم Interpreter يأخذ ما تكتبه، يحلله، وينفذه. واجهة الأوامر سطر Terminal هي عبارة عن تطبيق رسومي يشغل الـ Shell بداخلة.

القاعدة الأولى: تركيبة الأمر السليمة

أي أمر في لينكس يتكون من ثلاثة أجزاء يفصل بينها مسافة Space:

تركيب الأمر في لينكس
Command
ls
+
Options
-a -l
+
Arguments
/home
ls -a -l /home
  1. الكلمة المفتاحية Command: اسم التطبيق المراد تشغيله (مثل ls أو cd).
  2. الخيارات Options: تتحكم في سلوك الأمر وتغيّر طريقة عمله.
    • Short Option: يُكتب بشرطة واحدة مثل -a أو -l.
    • Long Option: يُكتب بشرطتين مثل --help أو --all.
  3. المدخلات Arguments: هو الإدخال Input الذي سيعمل عليه الأمر (غالبا ما يكون مساراً لملف أو مجلد).
⚠️ أخطاء شائعة

نسيان وضع مسافة Space بين الأمر والخيارات والمدخلات سيؤدي للخطأ (مثال خاطئ: ls-a).

SECTION 3.4

القاعدة الذهبية والمسارات Paths

القاعدة الذهبية للمهندس

عندما ترید القیام بأي مھمة في الـ Terminal، اسأل نفسك دائماً سؤالین:

💡 القاعدة الذهبية — بالغة الأهمية

1. ما هو الأمر Command المناسب لهذه المھمة؟

2. كيف أستخدمه؟ (كيف أمرر الـ Options والـ Arguments له؟)

أنواع المسارات Paths

في لينكس، المدخلات Arguments التي تمررها للأوامر هي في أغلب الأحيان مسارات Paths، وهناك نوعان للمسارات:

📌 المسار المطلق Absolute Path

تعريفه: هو المسار الذي يبدأ دائماً من الجذر /.

مثال:

/ / home / techleef / desktop
📌 المسار النسبي Relative Path

تعريفه: هو المسار الذي يبدأ من الحالي المجلد الذي تقف فيه الآن (ولا يبدأ بـ /).

مثال: إذا كنت تقف في /home/techleef/ وأردت الدخول للسطح المكتب تكتب فقط:

desktop
الخاصية المسار المطلق Absolute Path المسار النسبي Relative Path
نقطة البداية الجذر / دائماً المجلد الحالي
يبدأ بـ / اسم مجلد أو . أو ..
يعتمد على الموقع الحالي لا نعم
مثال /home/techleef/desktop desktop

الملفات والمجلدات المخفية

📘 مفهوم: الملفات المخفية Hidden Files

أي ملف أو مجلد اسمه يبدأ بنقطة (مثل .config) هو ملف مخفي، ولن يظهره الأمر ls العادي.

الرمز المعنى الشرح
. Current Directory نسبي مسار يشير إلى الحالي المجلد.
.. Parent Directory نسبي مسار يشير إلى السابق / الأب المجلد.
✅ ماذا يجب أن تتذكر؟

عندما تستخدم أمر ls /home فأنت لا تمرر مجلداً للأمر، بل تمرر مساراً Path، والأمر ls ذكي بما يكفي ليتأكد هل هذا المسار موجود؟ وهل هو مجلد ليقرأ محتواه؟

⚠️ أخطاء شائعة

خلط المطلق والنسبي المسار سيؤدي للخطأ لأن النظام سيبحث عن desktop مباشرة تحت الجذر / وليس في مسارك الحالي.

مثال: كتابة cd /desktop وأنت تقصد مجلداً أمامك.

SECTION 3.5

الأوامر الأساسية في لينكس Basic Linux Commands

إليك أهم الأوامر الأساسية بناءً على فلسفة Unix (كل أمر يقوم بمهمة واحدة فقط ويقوم بامتياز):

الأمر pwd

  • الوظيفة: طباعة المسار الحالي الذي تقف فيه Print Working Directory.
  • الاستخدام: يعطيك دائماً المسار المطلق Absolute Path لمكانك الحالي.
Shell
pwd /home/techleef

الأمر ls [Option] [Path]

  • الوظيفة: عرض محتوى الممرر له المسار. إذا لم تمرر مساراً، سيعرض محتوى المجلد الحالي List.
  • خيارات مهمة:
    • ls -a: يعرض الملفات كافة بما فيها التي المخفية (التي تبدأ بنقطة) (All).
Shell
# عرض محتوى المجلد الحالي ls document image script.sh # عرض جميع الملفات بما فيها المخفية ls -a . .. document image script.sh .bashrc .config

الأمر cd [Path]

  • الوظيفة: الانتقال من مسار إلى آخر Change Directory.
  • أمثلة:
    • cd /home: انتقال بمسار مطلق.
    • cd desktop: انتقال بمسار نسبي.
    • cd ..: الرجوع للخلف خطوة (للمجلد الأب).
Shell
# الانتقال بمسار مطلق cd /home # الانتقال بمسار نسبي cd desktop # الرجوع للمجلد الأب cd ..

الأمر mkdir [Path]

  • الوظيفة: إنشاء مجلد جديد فارغ Make Directory.
  • ملاحظة: إذا كان المجلد مسبقاً موجوداً أو حاولت إنشاؤه في مسار لا تملك فيه صلاحية (مثل مباشرة تحت الـ Slash / بدون سلاش) سيظهر خطأ.
Shell
mkdir new_folder

الأمر touch [Path]

  • الوظيفة: إنشاء ملف نصي فارغ Text File.
Shell
touch myfile.txt

الأمر rm [Option] [Path]

  • الوظيفة: مسح ملف Remove.
  • خيارات مهمة:
    • rm -r [Directory]: إجباري لمسح المجلدات؛ لأنه يمسح المجلد وكافة محتویاته الداخلیة بشكل متسلسل (Recursive).
Shell
# مسح ملف rm myfile.txt # مسح مجلد وكامل محتوياته rm -r old_folder
⚠️ تحذير مهم

أمر rm -r لا يضع الملفات في سلة المحذوفات. المحذوفات تُحذف نهائياً. استخدمه بحذر شديد.

الأمر man [Command]

  • الوظيفة: عرض الكتيب الإرشادي Manual لأي أمر للتعرف على جميع خياراته Options وطريقة استخدامه.
  • بديل سريع: استخدام خيار المساعدة --help مع أي أمر (مثال: ls --help).
Shell
# عرض الكتيب الإرشادي الكامل man ls # بديل سريع للمساعدة ls --help
الأمر الوظيفة مثال
pwd طباعة المسار الحالي Print Working Directory pwd
ls عرض محتوى المجلد List ls -a
cd الانتقال بين المجلدات Change Directory cd /home
mkdir إنشاء مجلد جديد Make Directory mkdir new_folder
touch إنشاء ملف نصي فارغ touch file.txt
rm مسح ملف Remove rm -r folder
man عرض الكتيب الإرشادي Manual man ls

أمثلة عملية لفهم السلوك

نفترض أننا لدينا ملف example.txt ولدینا في مجلد techleef:

🚀 مثال عملي 1: نقل وتغيير الاسم

نريد نقل example.txt إلى مجلد lab الموجود داخل desktop وتغيير اسمه إلى t1.txt:

mv example.txt desktop/lab/t1.txt

هنا استخدمنا المسار النسبي، والأمر سينقل الملف ويغير اسمه في نفس الخطوة.

🚀 مثال عملي 2: نسخ مع النقطة

نريد أخذ نسخة من t1.txt الموجود في lab ووضعها في مجلدنا الحالي (techleef) بنفس اسمها:

cp desktop/lab/t1.txt .

هنا استخدمنا النقطة كوجهة، مما يعني "انسخ الملف وضعه في المجلد الذي أقف فيه الآن".

🧠 فهم عميق: النقطة . كوجهة

عند استخدام النقطة . كـ Destination في أمر cp، فأنت تخبر النظام: "الوجهة هي المجلد الحالي". الملف سيُنسخ بنفس اسمه الأصلي إلى المكان الذي تقف فيه حالياً. هذا نمط شائع جداً في العمل اليومي مع لينكس.

SECTION 3.7

لماذا نستخدم ./ لتشغيل البرامج والنصوص البرمجية Scripts؟

سؤال مهم: عندما نشغل أمراً نكتبه مباشرة مثل ls، ولكن لتشغيل سكربت (مثلاً قمنا بكتابته script.sh موجود في مجلدنا الحالي) نضطر لكتابة ./script.sh؟ لماذا؟

التفسير التقني

  1. عندما تكتب أمراً مثل cp، فإن الـ Shell يبحث عنه في المسارات المجهزة مسبقاً في النظام (مثل /bin و /usr/bin).
  2. لكن الشل لا يبحث في مجلدك الحالي لدواعي أمنیة وتنظیمیة.
  3. لذلك، إذا كان لديك برنامج تنفيذي Executable في مجلدك الحالي، يجب أن توجه الشل إلى مساره يدوياً لكي يبحث عنه في نظام المسارات.
  4. استخدمنا النقطة (التي تعني المجلد الحالي) ودمجناھا مع / لتكوين مسار نسبي:
    . / script.sh
  5. قم بتشغيل الملف حرفياً في هذا المجلد.
آلية بحث الـ Shell عن الأوامر

↓

الحل: المجلد الحالي ليس في $PATH، لذلك نستخدم ./script.sh لنخبر الشل بمكان الملف تحديداً
✅ ماذا يجب أن تتذكر؟

الرمز ./ ليس "أمر تشغيل"، بل هو مجرد مسار نسبي Relative Path يخبر الشل بدقة بمكان التنفيذي الملف. يمكنك تشغيل الملف نفسه بشكل طبيعي باستخدام مساره المطلق (مثال: /home/user/script.sh) وسيعمل.

  • ✓النقطة . تعني المجلد الحالي، والنقطتان .. تعني المجلد الأب.
  • ✓أمر mv يتصرف بشكل مختلف حسب حالة الوجهة: مجلد موجود (نقل)، مسار غير موجود (نقل + إعادة تسمية)، ملف موجود (استبدال).
  • ✓الرمز ./ يُستخدم لأن الـ Shell لا يبحث في المجلد الحالي تلقائياً لأسباب أمنية.
📌 تعريفات مهمة
  • Root File System: نظام تنظيم الملفات في لينكس الذي يبدأ من الجذر / وتتفرع منه كل المجلدات.
  • Shell: مترجم Interpreter يأخذ الأوامر النصية ويحللها وينفذها عبر الكيرنل.
  • Absolute Path: مسار يبدأ دائماً من الجذر / ويحدد موقع الملف بشكل كامل ودون غموض.
  • Relative Path: مسار يبدأ من المجلد الحالي ويعتمد على موقعك الحالي في الشجرة.
  • Hidden Files: ملفات أو مجلدات أسماؤها تبدأ بنقطة . ولا تظهر مع ls العادي.
⚠️ أخطاء شائعة
  • نسيان المسافة Space بين أجزاء الأمر: ls-a خطأ، الصحيح ls -a.
  • خلط المسار المطلق مع النسبي: cd /desktop سيبحث تحت الجذر مباشرة وليس في مسارك الحالي.
  • محاولة كتابة script.sh مباشرة بدون ./ — الشل لن يجده لأن المجلد الحالي ليس في $PATH.
  • استخدام rm لمجلد بدون -r — لن يعمل لأن rm العادي يتعامل مع الملفات فقط.
🧠 ملاحظات للمقابلات
  • سؤال: "لماذا لينكس يستخدم شجرة واحدة بينما Windows يستخدم عدة أقراص؟" — الإجابة: فلسفة Unix تؤمن بأن كل شيء هو ملف، والشجرة الموحدة تسهل إدارة الصلاحيات والتصmount.
  • سؤال: "لماذا ./ ضروري لتشغيل السكربتات؟" — الإجابة: أسباب أمنية، الـ Shell لا يبحث في المجلد الحالي ضمن $PATH لتجنب تنفيذ ملفات خبيثة.
  • سؤال: "ما الفرق بين /bin و /sbin؟" — /bin أوامر لكل المستخدمين، /sbin أوامر للـ root (إدارية).
💡 نصائح للحفظ
  • المسار المطلق: يبدأ بـ / = "مطلق" = من الأعلى (الجذر).
  • المسار النسبي: لا يبدأ بـ / = "نسبي" = من هنا (حيث أنا).
  • pwd: Print Working Directory = "أين أنا الآن؟".
  • cd: Change Directory = "أريد الذهاب إلى...".
  • ./: "هنا في هذا المجلد" = النقطة = مكانك الحالي.
  • ..: "للخلف" = نقطتان = المجلد الأب.
📝 نصائح للامتحانات
  • قد يُسأل عن الفرق بين المسار المطلق والنسبي مع أمثلة — تأكد من إعطاء أمثلة صحيحة.
  • قد يُطلب منك كتابة أمر كامل لنقل ملف وإعادة تسميته في خطوة واحدة — استخدم mv مع وجهة غير موجودة.
  • توقع سؤالاً عن وظيفة كل مجلد من /bin, /lib, /boot, /home.
  • قد يُسأل: "ماذا يحدث إذا كتبت ls /home؟" — الجواب: يمرر مساراً وليس مجلداً، والـ ls يتحقق من وجوده ثم يعرض محتواه.
