4.1
البنية الداخلية لنظام الملفات
File System Internals

لفهم كيف يخزن Kernel التطبيقات والملفات في وحدة التخزين Storage Device، استعان المدرب بتشبيه "الأرض الزراعية".

🚀 مثال تطبيقي — كيف يتم تخزين الملفات فعلياً في الديسك؟
تخيل أن وحدة التخزين هي مساحة أرض مقسمة إلى أجزاء صغيرة تسمى قطاعات Sectors أو Data Blocks. التصميم الذي ينظم كيفية الكتابة في هذه الأجزاء هو نظام الملفات File System.

ويتكون من ثلاثة عناصر رئيسية:

📋

بلوك سوبر

🏷️
Inode
Index / Information Node — نود الآي
هيكل بيانات يمثل أي ملف أو مجلد — يحتوي على تفاصيل الملف ومؤشرات لمواقع البيانات
📦
Data Blocks
بلوكس داتا

📦 Data Blocks — بلوكس داتا

هي المساحات الفعليية التي تُكتب وتُخزن فيها بيانات الملفات. كل بلوك له حجم معين.

📋 Super Block — بلوك سوبر

هو بلوك خاص جداً يتواجد في بداية وحدة التخزين.

📘 مفهوم — وظيفة الـ Super Block

وظيفته: يحتوي على المعلومات الوصفية Metadata الخاصة بنظام الملفات ككل، مثل:

  • نوع نظام الملفات
  • حجم الملفات
  • المساحة الفارغة
  • كيف يتعامل القرص مع Data Blocks
  • إلخ...

أولاً يقرأ الكيرنل هذا البلوك ليفهم أولاً...

🏷️ Inode — Index/Information Node — نود الآي
📌 تعريف — Inode
هو هيكل بيانات Data Structure يمثل أي ملف أو مجلد في لينكس.

كل ملف في لينكس يملك به خاص Inode فریداً رقماً Unique Number.

🧠 فهم عميق — محتوى الـ Inode

المحتوى: لا يحتوي على بيانات الملف بل على تفاصيل الملف والأهم:

  • نوعه
  • حجمه
  • صلاحياته
  • تاريخ إنشائه

المؤشرات: يحتوي على مؤشرات Pointers تدل على أماكن الـ Data Blocks التي توجد بها البيانات الحقيقية للملف.

✅
تتذكر؟ الملف في لينكس ليس كتلة واحدة، بل Inode يحتوي على معلومات الملف ويشير إلى Data Blocks متفرقة تحتوي على البيانات.
📄 فلسفة يونكس: "كل شيء ملف" — Everything is a file

المجلدات Directories في لينكس هي حتى عن عبارة ملفات لها Inode خاص.

🧠 فهم عميق — كيف يعمل المجلد؟

كيف يعمل المجلد؟ المجلد إذن هو ملف Inode نوعه "Directory" ببساطة هو فِهرس بأسماء الملفات المخزنة داخله وأرقام الـ Inodes الخاصة بها.

⚠️ خطأ شائع
الاعتقاد بأن المجلد Directory هو "صندوق" في الديسك. برمجياً، المجلد مجرد ملف نصي يحتوي على أسماء ملفات أخرى.

4.2
المتغيرات في الشل
Shell Variables

بما أن الـ Shell هو لغة برمجة Programming Language ومترجم Interpreter، فإنه يدعم إنشاء المتغيرات Variables لتخزين البيانات.

📘 مفهوم — كيف تعمل المتغيرات في الشل؟

لا يوجد نوع لها No Data Types — المتغيرات في الشل ليس لها نوع (لا int, float, string...)، كل شيء يُعامل مبدئياً كنص.

طريقة التعريف: نكتب اسم المتغير متبوعاً بعلامة التساوي = ثم القيمة Value.

⚠️ قاعدة جداً مهمة — المسافات

يجب ألا توجد أي مسافة Space قبل أو بعد علامة التساوي =.

✅ الكتابة الصحيحة
NAME=techleef
❌ الكتابة الخاطئة
NAME = techleef

لماذا؟ إذا تركت مسافة، الشل سيحلل السطور بناءً على المسافات Spaces، سيعتبر أن الكلمة الأولى NAME هي أمر Command، وأن علامة = هي خيار Option، وأن techleef هو معامل Argument. ولأنه لا يوجد أمر اسمه NAME، سيظهر لك خطأ!

✅
تتذكر؟ إياك ووضع مسافات حول علامة التساوي عند تعريف المتغيرات في سطر أوامر لينكس.
⚠️ خطأ شائع
التعود بمسافات على تعريف المتغير (مثل لغات البرمجة الأخرى C أو Python).
⚡ مراجعة سريعة
VAR=value
⬅ تعريف صحیح لمتغیر
VAR = value
⬅ خطأ (سیعتبر VAR أمرًا)
No Data Types
كل شيء يُعامل كنص
4.3
أوامر الطباعة
echo & printf

لدينا أمران أساسيان لطباعة المخرجات على الشاشة Standard Output:

🔊 echo [Text/Variable]
📘 مفهوم — وظيفة echo

الوظيفة: طباعة النصوص أو محتوى المتغيرات على الشاشة مع إضافة سطر جديد New line تلقائياً في النهاية.

الاستخدام: يُستخدم للطباعة البسيطة والسريعة.

Shell 
# مثال: سيطبع النص كما هو
echo hello world
🖨️ printf [Format] [Arguments]
📘 مفهوم — وظيفة printf

الوظيفة: يتيح لك تنسيق المخرجات Formatting بدقة. أمر مطور يشبه دالة printf في لغة C.

الاستخدام: يُستخدم عندما تحتاج للتحكم الدقيق في شكل الطباعة أو إدخال الرموز الخاصة مثل \n للنزول لسطر جديد أو \t للمسافات.

Shell 
# مثال: سيطبع hello ثم ينزل لسطر جديد
printf "%s\n" hello
✅
تتذكر؟ printf أقوى لضمان التنسيق في كتابة السكربتات المعقدة، بينما echo أسهل للاستخدام اليومي.
4.4
الرمز الخاص ($) واستخداماته المتقدمة
Advanced $ Usage

الـ Shell يقوم بتحليل Interpret أي سطر تكتبه حرفياً. إذا كتبت echo NAME فسيطبع الشل كلمة "NAME" حرفياً. لإخبار الشل بأن هذه الكلمة ليست نصاً عادياً بل متغيراً يجب جلب قيمته، نستخدم رمز الدولار $.

1 الشل يقرأ السطر
↓
2 يجد رمز $ ويدرك أن ما بعده هو معامل (متغير خاص أو أمر)
↓
3 يذهب إلى الذاكرة الداخلية الخاصة ويبحث عن متغير بهذا الاسم
↓
4 يمسح كلمة المتغير من السطر ويستبدلها بالقيمة الحقيقية Substitution
↓
5 يمرر السطر النهائي إلى الـ Kernel لتشغيله
جدول الاستخدامات المتقدمة لرمز الدولار $
التنسيق Syntax الميزة / الوظيفة Advantage مثال Example
$VAR استخدام أو طباعة القيمة المباشرة للمتغير echo $NAME
${VAR}
  1. رفع الغموض Ambiguity: يفصل اسم المتغير عما بعده من حروف
  2. التعدیل Substitution: يمكن استخدامه لتوفير قيمة افتراضية Default Value إذا لم يكن المتغير موجوداً
 echo ${NAME}iable
echo ${PLACE:-Tunisia}
$(Command) أمر داخل أمر Command Substitution: الشل يقوم بتنفيذ الأمر الموجود بين الأقواس أولاً، ويأخذ نتيجتها، ويضعها مكان هذا المقطع لتشغيل الأمر الخارجي echo $(pwd)
يطبع مسار المجلد الحالي
✅
تتذكر؟ الأقواس المتعرجة {} تستخدم لنصوص المتغيرات والحماية من التداخل، بينما الدائریة الأقواس () تستخدم لتشغیل الأوامر وتمریر نتائجھا.
⚠️ أخطاء شائعة
  • الخلط بين القوس المتعرج والدائري: $(VAR) سيعتبر VAR أمراً ويحاول تشغيله مما ينتج عنه خطأ، بينما ${VAR} للمتغيرات الصحيحة.
  • استخدام علامة الاقتباس الفردية ' ': سيعطل الرمز $ ويطبع كنص عادي. استخدم دائماً علامات الاقتباس المزدوجة " " مع المتغيرات.
4.5
تشغيل عدة أوامر معاً
Command Separators

هل يمكنك تشغيل أكثر من أمر في سطر واحد؟ نعم، خلال استخدام رموز فصل خاصة تخبر الـ Shell بكيفية التعامل مع تسلسل الأوامر Execution Flow.

جدول أدوات ربط الأوامر
الرمز الاسم Name الفكرة الوظيفة مثال يوضح
; Semicolon
الفاصلة المنقوطة		 التنفيذ المتتالي الأعمى ينفذ الأمر الأول ثم الثاني، بغض النظر عن نجاح أو فشل الأمر الأول cmd1 ; cmd2
&& Logical AND
العامل المنطقي AND		 التشغيل المشروط (بالنجاح) ينفذ الأمر الثاني فقط إذا نجح الأمر الأول بدون أخطاء mkdir lab && cd lab
يدخل المجلد (لو فشل الإنشاء لن يدخل)
|| Logical OR
العامل المنطقي OR		 النسخة الاحتياطية (بالإخفاق) ينفذ الأمر الثاني فقط إذا فشل الأمر الأول cd lab || echo "Directory not found"
📊 مقارنة بصرية — سلوك فواصل الأوامر الثلاثة
;

cmd1
↓
cmd2
دائماً ينفذ الثاني
&&
Conditional (Success)
cmd1 ✅
↓
cmd2 ✅
فقط إذا نجح الأول
||
Fallback (Failure)
cmd1 ❌
↓
cmd2 ⚡
فقط إذا فشل الأول
✅
تتذكر؟ رمز && ممتاز في السكربتات لتجنب الكوارث؛ مثلاً لا تقم بنسخ الملفات إلا إذا تأكدت من نجاح إنشاء المجلد.
⚠️ خطأ شائع
استخدام الفاصلة المنقوطة ; لربط أوامر يعتمد بعضها على بعض، فلو فشل الأمر الأول سيكمل النظام تشغيل الثاني مما قد يؤدي إلى حذف أو نقل ملفات بالخطأ.
4.6
حالة الخروج
Exit Status / Return Value

كيف يعرف الـ Shell ما إذا كان الأمر الأول قد نجح أم فشل (كما في رمز &&)؟ السر يكمن في ما يُعرف بـ Exit Status.

📌 تعريف — ما هو الـ Exit Status؟

عندما تُبرمج تطبيقاً بلغة C وتنتهي دالة main بـ return 0، فإن هذا الصفر يُرسل إلى الـ Kernel والـ Kernel يرسله للـ Shell.

  • الرقم 0 يعني أن التطبيق اشتغل بشكل سلیم تماماً Success / No Errors
  • أي رقم غير 0 (مثال 1, 2, 127) يعني أن هناك خطأ Error قد حدث أثناء التشغيل (كل رقم يعبر عن خطأ محدد — مثلاً 127 يعني Command Not Found)
🔢 مخطط حالة الخروج — Exit Status Values
0
SUCCESS
No Errors
1
General Error
2
Misuse
127
Not Found
1 – 255 → أي رقم آخر = فشل / خطأ
⚙️ المتغير السري $?

بما أن الـ Shell افتراضياً لا يطبع حالة الخروج للمستخدم، فإنه يقوم فوراً بتخزين الرقم هذا في متغير داخلي خاص يحمل اسم علامة الاستفهام $?.

📘 مفهوم — وظيفة المتغير $?

الوظيفة: طباعة حالة الخروج Exit Status لآخر أمر تم تنفيذه.

Shell 
echo $?
💡 طريقة الاستخدام
تقوم بتشغيل أي أمر (مثال: ls) ثم تكتب فوراً echo $? — إذا طبع 0 فالأمر السابق تم بنجاح.
✅
تتذكر؟ يجب فحص $? فوراً بعد الأمر الذي ترید التأكد من حالته. یُحدَّث بعد كل أمر كأنه متغیر باسم غریب (علامة استفھام) لا تنشغل به حالتھ.

📘 المفاهيم الرئيسية — Main Concepts
  • نظام الملفات File System يتكون من ثلاثة عناصر: Data Blocks و Super Block و Inode
  • الـ Inode لا يحتوي على بيانات الملف بل على معلوماته ومؤشرات لمواقع البيانات الحقيقية
  • المجلدات في لينكس هي ملفات من نوع "Directory" تحتوي على فهرس بأسماء الملفات وأرقام الـ Inodes الخاصة بها
  • المتغيرات في الشل ليس لها أنواع بيانات No Data Types — كل شيء يُعامل كنص
  • لا مسافات حول علامة التساوي = عند تعريف المتغيرات
  • echo للطباعة البسيطة، printf للتنسيق الدقيق
  • رمز $ يستدعي قيمة المتغير، ${} للحماية والتعدیل، $() لتشغیل الأوامر
  • ; تنفيذ أعمى، && مشروط بالنجاح، || مشروط بالفشل
  • حالة الخروج Exit Status: 0 = نجاح، أي رقم آخر = خطأ
  • المتغير $? يحفظ حالة الخروج لآخر أمر تم تنفيذه
📌 تعريفات مهمة — Important Definitions
  • Data Blocks: المساحات الفعليية التي تُكتب وتُخزن فيها بيانات الملفات، كل بلوك له حجم معين
  • Super Block: بلوك خاص جداً يتواجد في بداية وحدة التخزين، يحتوي على Metadata خاصة بنظام الملفات ككل
  • Inode: هيكل بيانات Data Structure يمثل أي ملف أو مجلد في لينكس، يحتوي على تفاصيل الملف ومؤشرات Pointers لأماكن البيانات
  • Exit Status: رقم يُرسل من التطبيق إلى الـ Kernel ثم إلى الـ Shell يدل على نجاح أو فشل الأمر
  • Command Substitution: تشغيل أمر داخل أمر باستخدام $(Command) واستبدال نتيجته مكان المقطع
⚠️ أخطاء شائعة — Common Mistakes
  • الاعتقاد بأن المجلد Directory هو "صندوق" في الديسك — برمجياً هو مجرد ملف نصي فِهرسي
  • وضع مسافات حول = عند تعريف المتغيرات — نتيجة التعود من لغات مثل C أو Python
  • الخلط بين ${VAR} (متغير) و $(VAR) (أمر) — الثاني يُسبب خطأ
  • استخدام علامة الاقتباس الفردية ' ' مع المتغيرات — تعطل $
  • استخدام ; لربط أوامر مت-dependency — قد يُسبب حذف أو نقل ملفات بالخطأ
  • فحص $? بعد أكثر من أمر — يُحدَّث بعد كل أمر لذا يجب فحصه فوراً
🎯 ملاحظات المقابلات — Interview Notes
  • ما الفرق بين Inode و Data Block؟ — Inode يحتوي على معلومات الملف ومؤشرات، Data Block يحتوي على البيانات الفعلية
  • لماذا "كل شيء ملف" في يونكس؟ — حتى المجلدات هي ملفات من نوع Directory
  • ما الفرق بين echo و printf؟ — echo يضيف سطر جديد تلقائياً، printf يتيح تحكم دقيق في التنسيق
  • متى تستخدم ${VAR} بدلاً من $VAR؟ — عند الحاجة لفصل اسم المتغير عن النص المجاور أو استخدام قيم افتراضية
  • ما معنى Exit Status قيمته 127؟ — Command Not Found
  • لماذا && أفضل من ; في السكربتات؟ — لتجنب تنفيذ أوامر تعتمد على نجاح أمر سابق
🧠 حيل الذاكرة — Memory Tips
  • File System ثلاثية: تخيل أرشيف مكتبة — Super Block = فهرس الأرشيف، Inode = بطاقة تفاصيل الكتاب، Data Block = الرف الفعلي الذي عليه الكتاب
  • المسافات حول =: تذكر أن الشل يقرأ المسافات كفواصل بين أجزاء الأمر — أي مسافة تُفسّر الكلمة التالية كمعامل جديد
  • $ المتغيرات: ${} مثل قوسين يحمان الاسم، $() مثل آلة تنفيذ الأوامر
  • فواصل الأوامر: ; = "لا أهتم" (ينفذ دائماً)، && = "فقط إذا نجح"، || = "فقط إذا فشل" (الخطة ب)
  • Exit Status: صفر = كل شيء تمام (كعلامة ممتاز)، أي شيء آخر = مشكلة
📋 نصائح الامتحان — Exam Tips
  • توقع سؤال عن الفرق بين $VAR و ${VAR} و $(VAR)
  • احفظ قاعدة "لا مسافات حول =" — سؤال شائع في الامتحانات العملية
  • كن جاهزاً لكتابة أمر يستخدم && أو || بشكل صحيح
  • افهم الفرق بين echo و printf من حيث التنسيق والسطر الجديد التلقائي
  • توقع سؤال "ماذا يطبع echo $? بعد أمر ناجح؟" — الجواب: 0
  • كن قادراً على شرح عناصر نظام الملفات الثلاثة ووظيفة كل منها
